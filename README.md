## 대회 개요

**YelpZip 원본 데이터셋을 활용하여 GNN 기반 조직적 어뷰징 네트워크 탐지 모델을 구축하고, 이를 대시보드로 시각화하는 것**이 본 대회의 핵심입니다.

단순히 "이 글이 가짜인가?"를 판단하는 수준을 넘어, "이 리뷰들이 어떤 커넥션을 형성하고 있는가?"를 그래프 구조로 증명해 내는 것을 목표로 합니다.

---

## 기획 의도

기존의 텍스트 분석(NLP)만으로는 사기 리뷰를 완벽히 탐지하는 데 한계가 있습니다. GNN을 도입해야 하는 이유는 다음과 같습니다.

**① 개별 데이터가 아닌 조직적 패턴 포착**

사기 리뷰는 단순히 텍스트의 문제가 아니라, 특정 세력이 조직적으로 움직이는 네트워크의 문제입니다. GNN은 사용자·시간·장소를 기반으로 형성된 복잡한 연결망을 한눈에 파악하여, 겉보기에 정상적인 리뷰 뒤에 숨겨진 사기 네트워크를 찾아낼 수 있습니다.

**② 정보의 전파를 통한 정확도 향상**

GNN의 핵심 원리는 주변 정보의 전파입니다. 의심 노드와 연결된 주변 노드들의 정보를 함께 학습하기 때문에, 단독 데이터만 볼 때보다 훨씬 높은 정확도로 사기 집단을 식별할 수 있습니다.

**③ 변칙적인 수법에 대한 유연한 대응**

엣지(Edge)를 어떻게 설계하느냐에 따라 새로운 사기 수법도 효과적으로 방어할 수 있습니다. 텍스트를 바꾸더라도, 특정 계정을 돌려쓰거나 특정 상품에 집중하는 구조적 행동 패턴까지 숨기기는 매우 어렵기 때문입니다.

## 데이터셋

### (1) YelpZip — 메인 데이터셋

- **출처**: [Kaggle - YelpZip](https://www.kaggle.com/datasets/vaibhavsonkar/yelpzip)
- **규모**: 약 60만 개 이상의 식당·비즈니스 리뷰
- **활용 목적**: 텍스트를 NLP 모델로 임베딩하여 그래프의 노드로 구성하고, 대시보드 시각화 단계에서 사기 증거 자료로 활용

| 컬럼 | 설명 |
|------|------|
| `text` | 리뷰 원본 텍스트. BERT 등으로 임베딩하여 노드 피처로 활용 |
| `user_id` | 리뷰 작성자 고유 식별자. 동일 유저 리뷰 연결(R-U-R 엣지)에 활용 |
| `prod_id` | 대상 식당·상품 고유 식별자. 동일 식당 리뷰 연결에 활용 |
| `label` | 타겟 변수. **반드시 사기(-1)→1, 정상(1)→0으로 변환 필수** |
| `date` | 리뷰 작성 날짜. 특정 기간 집중 리뷰 탐지(R-T-R 엣지)에 활용 |
| `rating` | 별점 (1~5점). 노드 추가 피처 또는 동일 별점 연결(R-S-R 엣지)에 활용 |
| `tag` | 추가 메타 태그. 보조 데이터로 활용 가능 |

### (2) 참고용 데이터셋

출처: [CARE-GNN/data](https://github.com/YingtongDou/CARE-GNN/tree/master/data)

모델 학습에 직접 사용하지 않으며, 엣지 설계 아이디어를 참고하기 위한 벤치마크 데이터입니다.

**YelpChi (시카고 지역 식당·호텔 리뷰)**

- 노드: 리뷰(Review) / 정답지: 0(정상), 1(스팸)

| 엣지 | 설명 |
|------|------|
| R-U-R (Review-User-Review) | 같은 사용자가 작성한 리뷰들끼리 연결 |
| R-T-R (Review-Time-Review) | 같은 식당에 같은 달에 작성된 리뷰들끼리 연결 |
| R-S-R (Review-Star-Review) | 같은 식당에 같은 별점을 준 리뷰들끼리 연결 |

**Amazon (아마존 악기 카테고리 리뷰)**

- 노드: 사용자(User) / 정답지: 0(정상 유저), 1(사기꾼 유저)

| 엣지 | 설명 |
|------|------|
| U-P-U (User-Product-User) | 동일 제품에 리뷰를 남긴 사용자들끼리 연결 |
| U-S-U (User-Star-User) | 동일 제품에 같은 별점을 준 사용자들끼리 연결 |
| U-V-U (User-Vocab-User) | 리뷰 텍스트 유사도(TF-IDF)가 상위 5% 이내인 사용자들끼리 연결 |

## 참가자 가이드라인

### 필수 사항 (미준수 시 감점)

**1. 샘플링 및 분할 규정**

- 단순 무작위 추출 금지: 노드 간 연결성을 파괴하여 GNN 학습이 불가능해집니다.
- 밀도 중심 샘플링으로 **1만~5만 개** 사이의 노드를 추출하세요.

| 전략 | 설명 |
|------|------|
| 전략 A | 리뷰가 집중된 상위 N개 식당(Product) 중심 추출 |
| 전략 B | 왕성하게 활동하는 상위 N명 유저(User) 중심 추출 |
| 전략 C | 특정 연도·월 등 타임라인(Time-window) 중심 추출 |

- 데이터 분할: 샘플링 완료 후 학습용 80% / 시험용 20%로 분할
- 재현성 확보: 분할 시 사용한 코드와 `random_state` 등을 보고서에 명시

**2. 데이터 가공 규정**

원본 `label` 컬럼을 반드시 아래와 같이 변환 후 진행해야 합니다.

- 사기(-1) → 1
- 정상(1) → 0

**3. GNN 모델 사용 필수**

- 반드시 GNN 계열 모델(GCN, GAT, GraphSAGE 등)을 핵심으로 사용해야 합니다.
- 단순 머신러닝·딥러닝 모델만 단독 사용 시 큰 감점이 부여됩니다.

**GNN 모델 구조 설계 조건**

- **노드(Node)**: 리뷰(Review) 단위로 고정
- **엣지(Edge)**: 기본 Relation 1개 + 커스텀 Relation 1개 이상 필수

| 구분 | 조건 |
|------|------|
| 기본 Relation (택 1 필수) | R-U-R (동일 사용자), R-T-R (동일 기간), R-S-R (동일 별점) 중 하나 이상 포함 |
| 커스텀 Relation (창의성 평가) | 참가팀의 도메인 지식을 발휘한 창의적 엣지 설계. YelpChi, Amazon은 아이디어 참고용이며 독자적인 연결 방식과 파생 변수 설정을 적극 권장 |

## 보고서 양식

**규격**

- 용지: A4 / 본문 글자 크기: 11pt 고정 / 분량: 5~20 페이지 (표지·목차 제외)

**구성**
```
모델링 목표 정의
데이터 구성 및 전처리 과정
2-1. 사용 데이터
2-2. 전처리 내용
2-3. 파생 변수(Feature) 및 그래프 엣지(Edge) 설계 근거
모델 선택 이유와 적용 과정
3-1. 선택한 모델
3-2. 모델 선택 이유
3-3. 모델 적용 과정
모델 성능 평가 & 결과 해석
4-1. 평가 지표
4-2. 모델 성능 결과 (반드시 PR-AUC와 Macro F1 점수 포함)
4-3. 결과 해석
모델 기반 인사이트 및 활용 방안
모델링의 한계와 보완 계획
참고 코드 (선택)
추가 참고 자료 (선택)
```
